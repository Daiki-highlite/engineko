# ENGINEKO 会員コミュニティ — 運営プラットフォーム検討

> 作成日：2026-05-13
> 参照：会員コミュニティ_事例リサーチ.md

## 評価の前提：ENGINEKOが求める要件

プラットフォーム選定の前に、ENGINEKOのブランド哲学・事業フェーズから「何が必要で、何が不要か」を整理する。

### 必須要件

| 要件 | 理由 |
|------|------|
| **購入者のみ入会できる制御** | オープンコミュニティはブランド純度を損なう |
| **静かな通知設計** | 「押さない」ブランド5原則と矛盾しない運用が必要 |
| **Shopify / ECとの連携** | 購入履歴を起点にセグメントを切り、LTV計測に使う |
| **先行案内・限定情報の配信** | LTV最大化の核となる接点 |
| **日本語対応** | 国内オーナーが主体 |

### あると望ましい要件

| 要件 | 理由 |
|------|------|
| 英語対応・多言語 | 欧州展開フェーズ（Y2〜）で必要 |
| イベント管理機能 | 工房訪問・茶会・ポップアップの案内 |
| コンテンツアーカイブ | 窯入れライブの録画・コンセプトブック配布 |
| 運営工数が少ない | 小規模チームでも回せること |

### 不要な要件（採用しない）

| 不要な要件 | 理由 |
|-----------|------|
| ポイント・ランク機能 | ブランド哲学と矛盾する |
| パブリックなオープンコミュニティ | 希少性・純度を損なう |
| 高頻度のプッシュ通知 | 「押さない」5原則に違反 |
| 大規模スケール設計 | 質より量を求めない |

## 候補プラットフォーム 6選の評価

---

### 1. LINE公式アカウント ＋ LINEグループ

**概要**
- 日本のスマホユーザーの9割以上が使うメッセージアプリ。
- 公式アカウントでの一対多配信 ＋ LINEグループでのクローズドコミュニティの2層構造が取れる。
- LINEミニアプリでデジタル会員証・購入履歴連携も可能。

**ENGINEKOへの適合評価**

| 評価軸 | 評価 | コメント |
|--------|------|---------|
| 購入者限定制御 | ◎ | 購入後にLINE登録URLを案内する導線で制御可能 |
| 静かな通知設計 | ○ | 配信頻度を自分で制御できる。ただしLINEというメディアの性質上「通知が来るもの」という印象がある |
| Shopify連携 | ◎ | SocialPLUS等のアプリで購入データと連携可能 |
| 国内対応 | ◎ | 国内最強。開封率70%超（メールの3〜5倍） |
| 欧州対応 | × | LINEは欧州では普及していない |
| イベント管理 | △ | メッセージ配信のみ。予約フォームは外部連携必要 |
| コンテンツアーカイブ | × | チャットが流れていく構造でアーカイブに不向き |
| 運営工数 | ◎ | 運用が最もシンプル |
| コスト | ○ | 月額0〜数千円（配信数による） |

**強み**
- 国内オーナーとの距離感が最も近い
- ブランドがコントロールしやすいクローズド環境
- 購入後のウェルカムメッセージ自動化が可能（Shopify Flow連携）

**弱み**
- 欧州フェーズでは別チャネルが必要
- コンテンツが蓄積されない（アーカイブできない）
- LINEグループはオーナー同士の「場」としては閾値が低すぎる（誰でも気軽に入れる感）

---

### 2. Klaviyo（メール配信 ＋ CRM）

**概要**
- Shopifyと最も深く統合されているメールマーケティング ＋ CRMツール。
- 購入履歴・行動データをもとにセグメントを切り、パーソナライズされたメールを自動送信。
- コミュニティ「の場」ではなく、オーナーとの1対1の関係を管理する基盤。

**ENGINEKOへの適合評価**

| 評価軸 | 評価 | コメント |
|--------|------|---------|
| 購入者限定制御 | ◎ | Shopifyの購入データと完全同期。購入者のみをセグメントとして管理 |
| 静かな通知設計 | ◎ | 配信頻度を精緻にコントロール可能。メールは「受け取ったときに読む」メディア |
| Shopify連携 | ◎ | 業界最強。購入回数・金額・購入商品でセグメント管理 |
| 国内対応 | ○ | 日本語メール対応可。ただしUIは英語 |
| 欧州対応 | ◎ | 多言語・多通貨対応。欧州オーナーにも英語でアプローチ可 |
| イベント管理 | △ | メール告知のみ。予約は外部連携 |
| コンテンツアーカイブ | × | コミュニティの「場」ではない |
| LTV計測 | ◎ | 購入者のLTVをリアルタイムで可視化。コミュニティ参加者vs非参加者の比較も可 |
| 運営工数 | ○ | 初期設定に工数がかかるが、設定後はほぼ自動 |
| コスト | ○ | 無料〜$45/月（500件まで無料） |

**強み**
- LTV計測の基盤として必須。コミュニティの効果検証に不可欠
- 購入後のフロー（ウェルカム→工房ライブ案内→先行案内）を自動化できる
- 欧州オーナーにも同じ仕組みで対応できる

**弱み**
- コミュニティの「場」ではないため、オーナー同士の繋がりは生まれない
- 単体では「関係の継続」は作れても「縁の輪」は作れない

---

### 3. Circle.so

**概要**
- コース・イベント・ディスカッション・決済を一つにまとめたオールインワン型コミュニティプラットフォーム。
- 招待制のプライベートスペースを複数作れる。ブランド独自のコミュニティサイトとして機能。
- 料金：$89/月〜（Professional）、$199/月〜（Business）

**ENGINEKOへの適合評価**

| 評価軸 | 評価 | コメント |
|--------|------|---------|
| 購入者限定制御 | ◎ | スペースごとに招待制・非公開設定が可能 |
| 静かな通知設計 | ○ | 通知設定はメンバーが制御できる。プッシュよりメールベース |
| Shopify連携 | △ | 直接統合はない。Zapier等を介す必要あり |
| 国内対応 | △ | 日本語コンテンツは投稿可能だがUIは英語 |
| 欧州対応 | ◎ | 英語ネイティブのプラットフォーム。欧州オーナーに適合 |
| イベント管理 | ◎ | 専用のイベント機能あり。RSVP・ライブ配信も内包 |
| コンテンツアーカイブ | ◎ | 投稿・動画・ドキュメントが蓄積される |
| LTV計測 | × | Klaviyoとの連携が必要 |
| 運営工数 | ○ | 機能が多い分、初期設計に工数がかかる |
| コスト | △ | $89/月〜（機能追加で$188〜$277/月になりやすい） |

**強み**
- コンテンツ（工房ライブ録画・コンセプトブック）を蓄積・配布できる
- イベント（茶会・工房訪問の受付）と情報発信が一つのプラットフォームで完結
- 欧州オーナー向けのコミュニティとして機能しやすい

**弱み**
- 日本人オーナーにとって英語UIは障壁になりうる
- Shopifyとの直接連携がなく、購入者認証が手動になりがち
- コストが相対的に高い（Y1フェーズでは重い）

---

### 4. Discord

**概要**
- ゲーマー向けに生まれたリアルタイムチャット ＋ 音声コミュニティプラットフォーム。
- チャンネル分けと権限管理が細かく設定できる。

**ENGINEKOへの適合評価**

| 評価軸 | 評価 | コメント |
|--------|------|---------|
| 購入者限定制御 | ○ | ロール設定で入会制御は可能。ただし手動認証が必要 |
| 静かな通知設計 | × | リアルタイム通知文化が「押さない」哲学と根本的に矛盾 |
| ブランドトーン | × | Discord自体のUIがブランドイメージと合わない |
| 欧州対応 | ◎ | 欧州・北米のユーザーには普及 |
| 運営工数 | × | 24時間モデレーションが実質必要。荒らし・詐欺リスクも常態的に存在 |

**結論：採用しない**

高級ブランドはDiscordから離れる傾向にある（Luxe Daily調査）。理由は「Discordはショーウィンドウではなくパブリックスペース」であり、ラグジュアリーの本質である「アクセスの閾値を演出する」設計と矛盾する。ENGINEKOの「押さない、消さない」哲学とも根本的に合わない。

---

### 5. Geneva

**概要**
- Z世代向けの新興チャット型コミュニティアプリ。ミニマルなUIで「Discord よりも洗練されている」と評される。
- 無料で使えるが、有料メンバーシップ機能がない。

**ENGINEKOへの適合評価**

| 評価軸 | 評価 | コメント |
|--------|------|---------|
| 購入者限定制御 | △ | 招待制は可能。有料認証はできない |
| 静かな通知設計 | ○ | Discordより落ち着いたUX |
| コスト | ◎ | 無料 |
| 決済・LTV計測 | × | 収益機能なし。Klaviyoと繋がらない |

**結論：補助的利用のみ検討**

無料・UI良好だが、会費管理やLTV計測ができない。欧州フェーズで補助的に試すことはできるが、主軸にはなれない。

---

### 6. Substack（ニュースレター型）

**概要**
- メール配信＋有料購読の仕組みを持つニュースレタープラットフォーム。
- 「JOURNAL」として、購入者向けの読み物を届けるチャネルとして機能しうる。

**ENGINEKOへの適合評価**

| 評価軸 | 評価 | コメント |
|--------|------|---------|
| 購入者限定制御 | △ | 有料購読者限定配信は可能。ただし購入者かどうかの認証はできない |
| 静かな通知設計 | ◎ | メール配信。受け取ったときに読む文化 |
| コンテンツ蓄積 | ◎ | 記事がアーカイブされる |
| ブランドトーン | ◎ | 長文・思想系コンテンツに最も合う媒体 |
| LTV計測 | × | Klaviyoとは連携しない |
| コスト | ◎ | 無料〜（有料購読の10%を手数料として取る） |

**結論：コンテンツ配信の補助チャネルとして検討**

Klaviyoとは別に、ENGINEKOの「JOURNAL」（ブランド哲学・長文コンテンツ）をSubstackで配信し、オーナー向けの読み物コンテンツとして活用する余地がある。ただしコミュニティ管理の主軸にはなれない。

## ENGINEKOへの推奨構成

単一プラットフォームで全要件を満たすものは存在しない。**役割を分担した2〜3層の構成**が最適解。

---

### 推奨構成：Klaviyo（基盤）＋ LINE（国内接点）＋ Circle.so（欧州フェーズ以降）

```
【Y1フェーズ：国内立ち上げ】

  購入
    │
    ▼
  Klaviyo ─── LTVの計測・購入者データの管理・フロー自動化（基盤）
    │
    ├──▶ メール配信（先行案内・工房ライブ告知・JOURNAL）
    │
    └──▶ LINE公式アカウント ─── 国内オーナーとの日常的な接点
               │
               └──▶ LINEグループ（招待制） ─── オーナー同士の縁の場

【Y2フェーズ：欧州展開】

  Circle.so を追加
    │
    ├──▶ 英語コンテンツのアーカイブ（工房ライブ録画・概念ブック）
    ├──▶ イベント管理（欧州オーナー向けオンラインイベント）
    └──▶ Klaviyo と連携してLTV計測を継続
```

---

### 各プラットフォームの役割分担

| プラットフォーム | 役割 | 使うタイミング |
|----------------|------|--------------|
| **Klaviyo** | 購入者データの管理・LTV計測・自動フロー | 即時（Shopify立ち上げと同時） |
| **LINE公式アカウント** | 国内オーナーへの先行案内・イベント告知 | Y1ローンチと同時 |
| **LINEグループ（招待制）** | オーナー同士の縁の場（少人数・静か） | 最初の10名が集まったら開設 |
| **Circle.so** | コンテンツアーカイブ・欧州オーナーの場 | Y2（欧州展開フェーズ）から |
| **Substack** | JOURNAL長文・思想コンテンツの配信 | SNS戦略v2と並走させるなら検討 |

### Klaviyo 自動フロー設計（購入後のオーナー体験）

```
購入完了
  │
  Day 0 ─── ウェルカムメール
  │          「ENGINEKOがあなたの空間に届きました」
  │          + LINE登録への案内（コミュニティへの招待）
  │
  Day 3 ─── ブランドコンセプトメール
  │          「縁（En）について」─ JOURNALの世界観
  │
  Day 14 ── 工房・職人紹介メール
  │          「富山工房から」─ R2シリーズの動画リンク
  │
  Day 30 ── 1230℃ Live 案内メール
  │          「来月の窯入れ、オーナー先行でご案内します」
  │
  Day 60 ── 先行案内メール（次のエディション）
             「次の縁が、生まれます」

以降：月次で次の窯・新色・イベント情報を配信
```

### LINEグループ 運営設計

| 項目 | 設計 |
|------|------|
| 入会条件 | ENGINEKOを購入した人のみ（招待URLはKlaviyo経由で送付） |
| 規模感 | 最大50〜100名程度（大きくしない） |
| 投稿頻度 | 月2〜3回（窯入れ前後・ポップアップ前） |
| 投稿者 | ブランド側のみ（掲示板ではなく、静かな通知の場） |
| オーナー発言 | 任意・強制しない（縁が生まれたときだけ） |
| トーン | 40字以内・事実のみ・絵文字なし（ブランド5原則を踏襲） |

## 採用しないプラットフォームと理由

| プラットフォーム | 採用しない理由 |
|----------------|--------------|
| Discord | 「押さない」哲学と矛盾。モデレーション負担大。高級ブランドの離脱傾向が顕著 |
| Geneva | 決済・LTV計測機能なし。会員管理の主軸になれない |
| Slack | ビジネスツールとしての印象が強く、ブランドトーンに合わない |
| 独自アプリ開発 | 開発コスト数百万円〜。Y1フェーズでは投資対効果が低すぎる |
