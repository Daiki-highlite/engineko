# Slack 支出・収入報告 → ダッシュボード自動反映 セットアップ手順

Slack で `/expense` コマンドを打つと Google Sheets の「支出」シートに自動記録され、
キャッシュフローダッシュボードに反映される仕組みのセットアップ手順です。

```
Slack（/expense コマンド）
   → Google Apps Script（doPost が受信・パース）
   → Google Sheets「支出」シートに1行追加
   → ダッシュボード（cashflow.html）が読み込んで表示
```

Slack フリープランで動作します（ワークフロービルダー不要）。

---

## 1. Google Sheets の準備

「支出」シートの 1 行目（ヘッダ）に以下の列を**追記**します（既存の A〜F 列はそのまま）。

| 列 | ヘッダ | 入る値 |
|----|--------|--------|
| A | 日付 | yyyy-MM-dd 形式 |
| B | 費用科目 | 下記11科目リストから |
| C | 内容 | 具体的に何の費用か |
| D | 金額 | 数値 |
| E | 支払者 | 氏名 |
| F | メモ | 証憑番号・取引先など |
| G | 区分 | `事業全体` or `商品個別` |
| H | 費用タイプ | 区分が事業全体のとき `イニシャル` or `ランニング` |
| I | 対象商品 | 区分が商品個別のとき商品名（ダッシュボードの商品名と一致させる） |
| J | 報告元 | Slack 経由なら自動で `Slack (ユーザー名)` が入る |

※ G〜J のヘッダは Slack から最初の報告が来たときに自動で書き込まれるので、手動追記は省略可。
※ **既存の支出行**は G〜I 列が空欄のままだと「未分類」と表示されます。

「収入」シートは既存のまま（A:日付 B:種別 C:内容 D:金額 E:メモ）。F 列に報告元が自動で入ります。

## 2. Apps Script の更新

1. スプレッドシートから **拡張機能 → Apps Script** を開く
2. エディタの内容をこのフォルダの `Code.gs` で**丸ごと置き換える**
3. **デプロイ → デプロイを管理 → （鉛筆アイコン）編集 → バージョン: 新バージョン → デプロイ**

> ⚠️ 「新しいデプロイ」を作ると URL が変わってしまいます。
> 必ず「デプロイを管理」から**既存デプロイの新バージョン**として更新してください。

デプロイ設定は既存どおり「実行ユーザー: 自分」「アクセスできるユーザー: 全員」のままで OK。

## 3. Slack アプリの作成（フリープランで可）

1. https://api.slack.com/apps → **Create New App** → **From scratch**
   - App Name: `ENGINEKO経理` など
   - Workspace: ENGINEKO のワークスペース
2. 左メニュー **OAuth & Permissions** → **Bot Token Scopes** に以下を追加
   - `chat:write`
   - `chat:write.public`
   - `commands`
3. 左メニュー **App Home** → Bot User の Display Name・Username を設定
4. 左メニュー **Slash Commands** → **Create New Command**
   - Command: `/expense`
   - Request URL: Apps Script のデプロイ URL（`https://script.google.com/macros/s/…/exec`）
   - Short Description: `支出を記録（ラベル付きで入力）`
5. 同様にもう1つ作成
   - Command: `/income`
   - Request URL: 同じ URL
   - Short Description: `収入を記録（ラベル付きで入力）`
6. 左メニュー **Install App** → **Install to Workspace** → 許可

> 💡 Slack は日本語のコマンド名を受け付けないため `/expense` `/income` で登録します。

### （任意）セキュリティ強化

Slack アプリの **Basic Information → App Credentials → Verification Token** をコピーし、
Apps Script の **プロジェクトの設定 → スクリプト プロパティ** に
`SLACK_VERIFICATION_TOKEN` = （コピーした値）を追加すると、
Slack 以外からの POST を拒否するようになります（未設定でも動作はします）。

## 4. 使い方

### 支出の記録

```
/expense 金額: 費用科目: 内容: 商品: 支払者: 日付: メモ:
```

| ラベル | 必須 | 説明 |
|--------|------|------|
| 金額 | ✅ | `4609` `¥4,609` `4609円` `18万` いずれも可 |
| 費用科目 | ✅ | 下記11科目リストから選ぶ |
| 内容 | ✅ | 具体的に何の費用か |
| 商品 | ✅ | モデル名 ／ 共通イニシャル ／ 共通ランニング のいずれか |
| 支払者 | 任意 | 省略時 = Slack の投稿者 |
| 日付 | 任意 | 省略時 = 今日。例 `6/10` |
| メモ | 任意 | 証憑番号・取引先など |

**「商品」ラベルの値と意味：**
| 値 | 意味 |
|----|------|
| モデル名（例 `ENGINEKO猫`） | その商品に紐づく個別費用 |
| `共通イニシャル` | 事業全体にかかる初期投資 |
| `共通ランニング` | 事業全体にかかる継続費用 |

**費用科目リスト（全11科目）：**
> 製造原価 ／ 梱包・資材費 ／ 人件費 ／ 広告宣伝費 ／ 外部イベント出展費 ／
> 送料・物流費 ／ 旅費交通費 ／ 外注費 ／ 知財・法務費 ／ 通信費 ／ 消耗品・雑費

※ 科目を追加したいときは、このリストとチャンネルのピン留めメッセージを更新するだけ。コード変更不要。

**例：**
```
/expense 金額:4609 費用科目:製造原価 内容:型費用追加分 商品:ENGINEKO猫
/expense 金額:2415 費用科目:梱包・資材費 内容:ダンボール代 商品:共通ランニング メモ:初回出荷分
/expense 金額:18万 費用科目:知財・法務費 内容:EU商標出願 商品:共通イニシャル 支払者:早川
/expense 金額:87890 費用科目:外部イベント出展費 内容:展示台一式（bolda） 商品:ENGINEKO猫 日付:6/10 メモ:bolda見積BYS20260610
```

### 収入の記録

```
/income 金額: 種別: 内容: 日付: メモ:
```

| ラベル | 必須 | 説明 |
|--------|------|------|
| 金額 | ✅ | |
| 種別 | ✅ | `売上` または `助成金` |
| 内容 | ✅ | 具体的に何の収入か |
| 日付 | 任意 | 省略時 = 今日 |
| メモ | 任意 | |

**例：**
```
/income 金額:50000 種別:売上 内容:ENGINEKO猫1体 メモ:〇〇様
/income 金額:300万 種別:助成金 内容:特別創業支援補助金
```

`/expense ヘルプ` または `/income ヘルプ` で書式リマインドが（自分にだけ）表示されます。
記録に成功するとチャンネルに「✅ 支出を記録しました」と投稿されます。

## 5. ダッシュボードへの反映

- 記録後、cashflow.html の **↻ 最新化** ボタンを押すか、ページをリロード
- 「商品別 損益」セクション: 商品ごとに 売上 −（商品個別支出）＝ 粗利 を表示
- 「事業全体 損益」カード: 全商品粗利 − 事業全体支出（ランニング・イニシャル別）
- 「支出実績」テーブル: 区分チップ表示＋区分フィルタボタン

## 6. 費用科目を追加したいとき

コードの変更は**不要**です。費用科目は自由記述なので、リスト外の科目名でも記録されます。

1. Slack チャンネルのピン留めメッセージ（使い方ガイド）の科目リストに追記
2. このファイルの科目リストにも追記（正本として管理）

> ⚠️ 手打ちで「外注費 / 外注 / 委託費」のように表記ゆれが起きると集計上は別科目になります。
> ピン留めメッセージからコピペする運用を徹底してください。

## トラブルシューティング

| 症状 | 原因と対処 |
|------|-----------|
| Slack で `timeout` エラー | Apps Script の初回起動が3秒を超えた。記録自体は成功していることが多いので Sheets を確認。頻発する場合は再実行で OK |
| 「認証エラー」と返る | スクリプトプロパティの `SLACK_VERIFICATION_TOKEN` と Slack アプリの Verification Token が不一致 |
| ダッシュボードに反映されない | デプロイを「新バージョン」で更新したか確認 |
| 商品別損益に出ない（⚠️商品名不一致） | `商品:〇〇` の商品名がダッシュボードの商品入力テーブルの商品名と一致していない |
| `/expense は有効なコマンドではありません` | Slack アプリが未インストール。Install App から再インストールし、Bot User を App Home で設定する |
