# Slack 支出報告 → ダッシュボード自動反映 セットアップ手順

Slack で `/expense` コマンドを打つと Google Sheets の「支出」シートに自動記録され、
キャッシュフローダッシュボードに反映される仕組みのセットアップ手順です。

```
Slack（/expense コマンド）
   → Google Apps Script（doPost が受信・パース）
   → Google Sheets「支出」シートに1行追加
   → ダッシュボード（cashflow.html）が読み込んで表示
```

Slack フリープランで動作します（ワークフロービルダー不要）。

---

## 1. Google Sheets の準備

「支出」シートの 1 行目（ヘッダ）に以下の列を**追記**します（既存の A〜F 列はそのまま）。

| 列 | ヘッダ | 入る値 |
|----|--------|--------|
| A〜F | 日付／カテゴリ／項目／金額／支払者／メモ | 既存のまま |
| G | 区分 | `事業全体` or `商品個別` |
| H | 費用タイプ | 区分が事業全体のとき `イニシャル` or `ランニング` |
| I | 対象商品 | 区分が商品個別のとき商品名（ダッシュボードの商品入力テーブルの商品名と一致させる） |
| J | 報告元 | Slack 経由なら自動で `Slack (ユーザー名)` が入る |

※ G〜J のヘッダは Slack から最初の報告が来たときに自動で書き込まれるので、手動追記は省略可。
※ **既存の支出行**は G〜I 列が空欄のままだとダッシュボード上で「未分類」と表示されます。
   さかのぼって区分を入力すると商品別損益・全体損益に正しく集計されます。

「収入」シートは既存のまま（A:日付 B:種別 C:項目 D:金額 E:メモ）。F 列に報告元が自動で入ります。

## 2. Apps Script の更新

1. スプレッドシートから **拡張機能 → Apps Script** を開く
2. エディタの内容をこのフォルダの `Code.gs` で**丸ごと置き換える**
3. **デプロイ → デプロイを管理 → （鉛筆アイコン）編集 → バージョン: 新バージョン → デプロイ**

> ⚠️ 「新しいデプロイ」を作ると URL が変わってしまいます。
> 必ず「デプロイを管理」から**既存デプロイの新バージョン**として更新してください。
> （URL が変わった場合は cashflow.html の `APPS_SCRIPT_URL` と Slack 側の Request URL を両方差し替え）

デプロイ設定は既存どおり「実行ユーザー: 自分」「アクセスできるユーザー: 全員」のままで OK。

## 3. Slack アプリの作成（フリープランで可）

1. https://api.slack.com/apps → **Create New App** → **From scratch**
   - App Name: `ENGINEKO 経理ねこ` など好きな名前
   - Workspace: ENGINEKO のワークスペース
2. 左メニュー **Slash Commands** → **Create New Command**
   - Command: `/expense`
   - Request URL: Apps Script のデプロイ URL（`https://script.google.com/macros/s/…/exec`）
   - Short Description: `支出を記録（金額 カテゴリ 項目 区分）`
   - **Escape channels…** はオフのまま
3. 同様にもう1つ作成
   - Command: `/income`
   - Request URL: 同じ URL
   - Short Description: `収入を記録（金額 種別 項目）`
4. 左メニュー **Install App** → **Install to Workspace** → 許可

> 💡 Slack は日本語のコマンド名を受け付けないため `/expense` `/income` で登録します。
> `/支出` `/収入` と打っても Code.gs 側で両方受け付けます。

### （任意）セキュリティ強化

Slack アプリの **Basic Information → App Credentials → Verification Token** をコピーし、
Apps Script の **プロジェクトの設定 → スクリプト プロパティ** に
`SLACK_VERIFICATION_TOKEN` = （コピーした値）を追加すると、
Slack 以外からの POST を拒否するようになります（未設定でも動作はします）。

## 4. 使い方

```
/expense 金額 カテゴリ 項目 区分 [支払者:名前] [日付:6/10] [メモ:自由記述]
```

- **金額**: `4609` `¥4,609` `4609円` `18万` いずれも可
- **カテゴリ**: 製造原価／梱包・資材費／人件費／広告宣伝費／送料・物流費／旅費交通費／外注費／知財・法務費／通信費／消耗品・雑費
- **区分**（必須・いずれか1つ）:
  - `イニシャル` … 事業全体にかかる初期投資
  - `ランニング` … 事業全体にかかる継続費用
  - `商品:商品名` … 特定商品にかかる費用（商品名はダッシュボードの商品名と一致させる）
- 省略時は 支払者=Slackの投稿者、日付=今日

**例**

```
/expense 4609 製造原価 型費用追加分 商品:ENGINEKO猫
/expense 2415 梱包・資材費 ダンボール代 ランニング メモ:初回出荷分
/expense 18万 知財・法務費 EU商標出願 イニシャル 支払者:早川
/income 50000 売上 ENGINEKO猫1体 メモ:〇〇様
/income 300万 助成金 特別創業支援補助金
```

`/expense ヘルプ` で書式リマインドが（自分にだけ）表示されます。
記録に成功するとチャンネルに「✅ 支出を記録しました」と投稿されるので、チームの誰でも見えます。

## 5. ダッシュボードへの反映

- 記録後、cashflow.html の **↻ 最新化** ボタンを押すか、ページをリロード
- 「商品別 損益」セクション: 商品ごとに 売上 −（商品個別支出）＝ 粗利 を表示
- 「事業全体 損益」カード: 全商品粗利 − 事業全体支出（ランニング・イニシャル別）
- 「支出実績」テーブル: 区分チップ表示＋区分フィルタボタン

## トラブルシューティング

| 症状 | 原因と対処 |
|------|-----------|
| Slack で `timeout` エラー | Apps Script の初回起動が3秒を超えた。**記録自体は成功していることが多い**ので Sheets を確認。頻発する場合は再実行で OK |
| 「認証エラー」と返る | スクリプトプロパティの `SLACK_VERIFICATION_TOKEN` と Slack アプリの Verification Token が不一致 |
| ダッシュボードに反映されない | デプロイを「新バージョン」で更新したか確認。URL が変わっていないかも確認 |
| 商品別損益に出ない（⚠️商品名不一致） | `商品:〇〇` の商品名がダッシュボードの商品入力テーブルの商品名と一致していない |
